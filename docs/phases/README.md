# Phase Detail Index

This directory contains one file per development phase of the Easy Packet Sniffer project. Each file has three required sections:

1. **Relevant knowledge** — concepts the developer must understand before writing code for this phase.
2. **Resources for learning and research** — books, RFCs, documentation, and reference implementations.
3. **Steps for implementation** — the concrete, ordered actions required to complete the phase.

Read `PROJECT_GUIDE.md` (in the project root) first for the overall context, then read these files in order as you progress.

| Phase | Title | Estimated Duration |
|---|---|---|
| 0 | [Foundations and Environment Setup](phase-00-foundations.md) | 1 week |
| 1 | [Terminal Capture Loop](phase-01-capture-loop.md) | 1 week |
| 2 | [Packet Parsing and Domain Object](phase-02-parsing.md) | 1–2 weeks |
| 3 | [Persistence (pcap Read/Write)](phase-03-pcap-io.md) | 3–5 days |

| 5 | [Minimum Viable PyQt6 UI](phase-05-ui-mvp.md) | 2 weeks |

| 7 | [Capture and Display Filters](phase-07-filters.md) | 1 week |
| 8 | [Statistics](phase-08-stats.md) | 1 week |
| 9 | [Production Hygiene](phase-09-hygiene.md) | 3–5 days |
| 10 | [Portfolio Presentation](phase-10-portfolio.md) | 3–5 days |

**Discipline:** Do not skip a phase. Each phase has a self-administered verification gate at the end. Pass the gate before moving on. If you cannot pass the gate, the previous phase is incomplete.
